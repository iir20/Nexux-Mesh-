# Kotlin Style Guide & Formatting Conventions

## Purpose
Enforces the coding style, layout patterns, indentation sizes, and naming schemes across all source files.

## Guidelines
- Indentation: 4 spaces.
- Variable Naming: `camelCase` (e.g., `peerNodeId`).
- UI Composable Methods: `PascalCase` (e.g., `SystemDiagnosticsPanel`).
